# API制限エラー対応機能 🚨

Chat.py v8.0.0で実装された包括的なAPI制限エラー対応機能について説明します。

## 🎯 概要

AIプロバイダーのAPI制限やトークン制限によって発生するエラーを自動的に検出し、適切な対応を行う機能です。ユーザーが手動で対応する必要がなく、システムが自動的に最適な解決策を実行します。

## 🔍 対応エラー一覧

### 📏 413 - Request too large（リクエットサイズ超過）

**原因:**
- 会話履歴が長すぎる
- 入力テキストが大きすぎる
- モデルのコンテキストウィンドウを超過

**自動対応:**
```
🔴 リクエストサイズエラー: Request too large
📊 現在の履歴サイズ: 約8,500文字
🔧 履歴を自動削減しました: 8,500 → 4,250文字
🔄 削減された履歴で再試行します...
```

**実装詳細:**
- 会話履歴を50%削減（最新のメッセージを優先保持）
- 削減前後のサイズを表示
- 削減後に自動的に再試行
- 最低限の履歴（2メッセージ）は保持

### ⏱️ 429 - Rate limit exceeded（レート制限超過）

**原因:**
- APIの1分間あたりの呼び出し回数制限
- トークン使用量の上限に達した
- 無料プランの制限

**自動対応:**
```
🟡 レート制限エラー: Rate limit exceeded
⏰ 60秒待機してから再試行します...
```

**実装詳細:**
- 指数バックオフによる待機時間計算
- 1回目: 60秒、2回目: 120秒、3回目: 240秒
- 待機時間の表示とカウントダウン
- 最大3回まで自動リトライ

### 🚫 503 - Service unavailable（サービス一時停止）

**自動対応:**
```
🟡 サービス一時停止: Service unavailable
⏰ 60秒待機してから再試行します...
```

### ⏰ 504 - Gateway timeout（ゲートウェイタイムアウト）

**自動対応:**
```
🟡 タイムアウトエラー: Gateway timeout
⏰ 30秒待機してから再試行します...
```

## 💾 トークン節約機能

### まとめエントリの除外
- CSVログのIsSummaryフラグで識別
- 会話履歴からまとめを自動除外
- summariesフォルダに別途保存

### 履歴サイズ管理
- 会話履歴のサイズ推定
- 413エラー時の自動削減（50%）
- 最新メッセージの優先保持

## 🎨 ユーザーエクスペリエンス

### 視覚的なエラー表示
- 🔴: 深刻なエラー（手動対応が必要）
- 🟡: 一時的なエラー（自動対応中）
- ⏰: 待機時間表示
- 🔧: 自動修復作業中
- 💡: 対応策の提示

### 対応策の提示
各エラーに対して具体的な解決策を表示します。

## 🎉 メリット

- **手動対応不要**: エラーが自動的に解決
- **継続的な会話**: 長時間の会話でも中断されない  
- **分かりやすい表示**: エラーの原因と対応状況が明確
- **システム安定性**: 予測可能で堅牢な動作

---

この機能により、ユーザーはAPI制限を気にすることなく、長時間の建設的な会話を継続できます。
